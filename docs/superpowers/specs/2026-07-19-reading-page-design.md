# 阅读页设计 spec

> 日期:2026-07-19
> 依据:`docs/product/2026-07-15-rhythm-product-discovery-handoff.md` 第 8 节 + 第 12 节
> 原型:`docs/product/prototypes/reading-a-shelf.html`
> 状态:brainstorm 已完成,方案 A · 书架为主 定稿

## 1. 目标与主用户场景

阅读页是底部导航第 4 个 tab。以书为主角,同时保留划线笔记的偶遇感与 AI 跨书理解。

打开阅读页,主用户会做:
1. **翻书找划线**——回顾在读书籍、点开划线笔记
2. **偶遇一句好话**——顶部随机词条(继承并升级现有 /couple 页的随机词条能力)
3. **看 AI 聚合的跨书主题**——不同书里相近的观点汇成"认知主题"
4. **从阅读到实践**——把书里的一个观点变成一轮实践(handoff 第 8.3 节)

## 2. 页面结构(书架优先)

自上而下 7 段。段之间的视觉密度不同,不是同构卡片列。

```
[顶部]  title 栏 + 微信读书同步按钮
[段 1] 随机词条卡(点换句,长按变议题,可收藏)
[段 2] 阅读汇总条(本周 h / 在读 n / 今年读完 n / 划线 n)
[段 3] 在读书籍 3D 书架横滑(带"添加书籍"占位)
[段 4] 词条流(按书分组,可折叠;每条支持"想试试")
[段 5] AI 跨书主题(横滑大卡,含来源书 + "试试" CTA)
[段 6] AI 推荐尝试(单卡,关联当前议题,三态按钮)
[段 7] 已读 / 暂停(紧凑列表)
[FAB]  纸书笔记(右下悬浮)
```

## 3. 各段详细设计

### 3.1 随机词条卡(顶部)

- 引文样式,`❝` 装饰,Noto Serif SC
- 右上角"换一句"按钮 → 循环显示 highlights 池
- 长按整卡 → 弹底表(Bottom Sheet)选择:"变为议题" / "关联到方向" / "收藏"
- 数据源:`reading_highlights` 表全部有 `mark_text` 的划线
- 如无数据:显示引导"去同步微信读书 →"

### 3.2 阅读汇总条

一行 4 项,横排等分。**只呈现累计数据,不呈现走势图**(走势图归"数据"深入页,不在首屏)。

- **本周时长**——本周所有 reading_sessions.duration_minutes 求和,以小时显示,保留一位小数
- **在读**——`reading_books.status = 'reading'` 的数量
- **今年读完**——`reading_books.status = 'finished'` AND 完成时间在当年
- **划线**——`reading_highlights` 表总数

### 3.3 在读书籍 3D 书架

- 横滑列表,`overflow-x: auto` + `scroll-snap`
- 每本书 3D 立体书脊效果(左侧阴影 + 内阴影),102px 宽 145px 高
- 卡片显示:书名、作者、进度%、划线数
- 最后一张是"添加书籍"占位(虚线边框 + 号)
- 数据源:`reading_books.status = 'reading'` 
- 点击某本 → 跳转单书详情页 `/reading/book/[id]`

**已完成的书不在此展示**——已完成书在段 7,进行中和已完成视觉分开。

### 3.4 词条流(按书分组)

- 每本书一张 r-card,head 显示书封 mini + 书名 + 划线条数 + 展开箭头
- 默认全部折叠;点击 head 展开该书全部划线
- 每条划线显示:
  - `mark_text`(原文)
  - `thought`(用户想法,如有)
  - "想试试"小按钮 → 跳到实践创建流程,预填假设 = 该划线
- 数据源:`reading_highlights` 按 `book_id` 分组

### 3.5 AI 跨书主题(横滑)

**新数据模型:** 需要新表 `reading_themes`,包含:
- `id`, `user_id`, `title`, `description`
- `source_book_ids` (jsonb array)
- `source_highlight_ids` (jsonb array)  
- `snippet_text`, `snippet_source`(展示的代表句)
- 生成时间 / 用户是否已确认

- 横滑 240px 宽的卡片
- 每卡显示:
  - kicker:"主题 · N 本书 M 条"
  - 主题名(Noto Serif SC)
  - 简短描述(1-2 行)
  - 来源书籍徽章(3 个以内)
- 点击卡片 → 主题详情页 `/reading/theme/[id]`,列出该主题下所有原始划线
- **AI 边界(handoff 第 9 节):** 主题只是索引,必须能回到原书原文,不覆盖用户原文

### 3.6 AI 推荐尝试

只显示 1 张(和"计划 · I2 · Do Next"里同款推荐卡的变体)。规则:
- 匹配当前进行中的议题 + 相关阅读划线
- 显示:
  - 一句推荐(粗体):具体假设 + 时长
  - 依据:引用哪些划线 / 与议题的匹配度
  - 三态按钮:「发起实践」/「先收藏」/「忽略」
- 数据源:`ai_recommendations` 表(和记录页周回顾的观察表共用)

### 3.7 已读 / 暂停

- 紧凑列表(高度 44px 一行),不占视觉篇幅
- 每行:书封 mini(24×32)+ 书名 + 状态(读完 / 暂停 45%)
- 数据源:`reading_books.status IN ('finished', 'paused')`
- 显示前 3-5 条 + "全部 N 本 →" 链接

### 3.8 纸书笔记 FAB

- 右下悬浮圆角胶囊
- 点击 → 打开新增纸书笔记页 `/reading/note/new`
- 表单字段:书名 / 作者 / 页码 / 原文 / 想法
- 保存后作为一条 `reading_highlights` 记录,`source = 'manual'`

## 4. 单书详情页(钻取)

路由:`/reading/book/[id]`

结构:
- 顶部返回栏
- 书封 hero(大)+ 书名 + 作者 + 状态
- 元数据:总页数 / 已读 / 划线数 / 加入时间
- **相关实践**(如果该书的划线曾创建过实践 → 显示)
- 划线列表(时间倒序)
- 阅读记录(sessions 时间线)

不做:笔记编辑器(现有笔记以只读+跳转到划线所在书为主)

## 5. AI 主题详情页(钻取)

路由:`/reading/theme/[id]`

结构:
- 顶部返回栏 + 主题名
- 主题描述(AI 生成)
- **原始划线列表**——按书分组,每条划线明确显示来源书籍
- 底部:"用这个主题发起实践"CTA(预填假设为主题描述)

**必须能回到原书**——每条原文可点跳转到对应 `/reading/book/[id]`。

## 6. 数据流

### 6.1 现有表(复用)
- `reading_books`:书籍
- `reading_sessions`:阅读时段
- `reading_highlights`:划线笔记(含 `source: weread | manual`)

### 6.2 新表
- `reading_themes`:AI 跨书主题(schema 见 3.5)
- `ai_recommendations`:AI 推荐尝试(与记录页周回顾共用)

### 6.3 用户交互对数据的影响
- 长按随机词条 → 变议题:创建新 `topic` 记录,关联到该 highlight
- 点"想试试" → 打开实践创建向导,预填假设
- 点"发起实践" → 直接创建 practice,状态=进行中
- 收藏词条 → 新增 `highlight_favorite` 关联
- 主题"确认" → theme 状态为 confirmed;"忽略" → dismissed

## 7. 视觉与交互约束

- 沿用 rhythm 冷调 token,4 皮肤可切
- 段之间视觉密度不同(引文卡 / 汇总条 / 书架 / 分组卡 / 横滑 / 单卡 / 紧凑列表 / FAB),避免"同构卡片"
- 3D 书脊效果不用图片,用 CSS 渐变 + 阴影模拟
- 长按交互移动端应支持 300ms 长按 + 触觉反馈(navigator.vibrate)
- 深色/浅色主题下书脊颜色可用不同色系(冷蓝 / 灰蓝 / 咖啡 / 青)让书架有识别度

## 8. 不在本次范围

- 微信读书 API 同步的后端实现(现有 reading-highlights.tsx 已有基础,复用即可)
- 拍照 OCR 加笔记(handoff 第 8.1 节说"后续可以研究")
- 阅读时间自动记录 / 与手环联动
- 分享划线到外部
- 情侣共享阅读(couple 模块已删)

## 9. 关联模块

- **计划页**:创建实践时,若假设来源=某条划线,记录字段 `source_highlight_id`
- **记录页**:实践详情页展示"本轮实践的书籍来源"(如有)
- **今天页**:如果实践的假设来自划线,今天页的实践卡可显示"来自《书名》"

## 10. 打开的问题(需在实现前决定)

1. AI 主题的生成周期?→ 推荐周日凌晨定时,与周回顾同批
2. AI 主题最少需要多少条相关划线才生成?→ 推荐 ≥ 3 条 且 来自 ≥ 2 本书
3. 词条流是否需要虚拟滚动?→ 100 条以内不需要;超过 300 条再考虑
4. 长按变议题的交互在桌面端怎么退化?→ 右键菜单 / 长按 400ms(mouse-hold)
