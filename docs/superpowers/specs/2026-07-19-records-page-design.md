# 记录页设计 spec

> 日期:2026-07-19
> 依据:`docs/product/2026-07-15-rhythm-product-discovery-handoff.md` 第 10、11、12 节
> 状态:brainstorm 已完成,等实施计划

## 1. 目标与主用户场景

记录页是底部导航第 3 个 tab。主用户是**正在跑实践的产品所有者本人**,在这里主要做两件事:

1. **回看实践证据**——查看进行中或已完成实践的所有记录,判断有没有效
2. **看趋势**——通过图表回顾实践完成率、睡眠时长、方法沉淀等长期变化

计划页展示"当前实践的当天/最近状态",记录页展示"所有实践的历史全集"。两者分工不重叠。

## 2. 顶层 tab 结构

保留顶部分段切换(和现有 records-page-client.tsx 的交互模式一致),4 个 tab,统一两字命名:

```
[ 记录  |  睡眠  |  复盘  |  回顾  ]
```

| tab | 内容 | 说明 |
|-----|------|------|
| 记录 | 所有实践列表 + 趋势图表 | 内部再分子 tab |
| 睡眠 | 只放分析图,无手动录入 | 等接入手环 |
| 复盘 | 今日复盘 + 检查点自动汇总 + 历史 | 融合每日复盘和实践检查点 |
| 回顾 | AI 周回顾全文 + 历史 | 单页滚动 |

删除:运动 tab、阅读 tab(阅读作为独立顶级模块,不属于记录页)。

## 3. "记录" tab 详情

### 3.1 子 tab 切换

```
[ 记录  |  趋势  ]
```

默认打开"记录"子 tab。

### 3.2 "记录"子 tab — 所有实践列表

- 按时间倒序,进行中的实践置顶,标"进行中"徽章
- 每条列表项显示:
  - 实践名称
  - 轮次标签(进行中 / 已完成)
  - 轮次日期范围(如 7月10日–7月16日)
  - 假设(进行中)或结论摘要(已完成)
  - **轮次进度用文字表达**:"第 2 轮 · 第 4 / 7 天"(进行中);"第 1 轮 · 完成"(已结束)
  - **不使用完成率百分比或进度条**——实践没有固定完成指标
- 点击某条实践 → 进入**实践详情页**

### 3.3 实践详情页(不是 tab,是钻取页)

顶部返回按钮 + 实践名称。

**顶部摘要区:**
- 本轮假设(1-2 行)
- 周期与进度(第 4 / 7 天)
- 若已结束:结论(有效 / 调整再试 / 放下)

**主内容:时间线**
- 每日记录从上到下按日期倒序
- 轻量记录:1 行(完成 / 做了一点 / 没开始 + 可选原因)
- 详细记录:展开卡片(原话、行动经过、想法与原因)
- 检查点用分隔线 + 小标题标记,展示自动汇总(3 天数据统计)+ 追问

**底部:本轮结论卡**
- 已结束轮次显示已确认结论
- 进行中轮次最后一天可写结论,系统生成候选,用户确认或修改

### 3.4 "趋势"子 tab

**不重复"记录"子 tab 的实践列表内容,仅展示图表。**

4 组独立分区,每组=眉标题 + 单独图表卡 + 底部一句结论描述:

1. **实践 · 完成率**——横向柱状图,每轮一条,标出轮次名称,当前轮次高亮
2. **方法沉淀**——SVG 面积折线图,x 轴按周,y 轴累计方法数,最新点强调
3. **睡眠 · 时长趋势**——手环接入前显示占位卡;接入后是 30/60/90 天折线图
4. **复盘 · 节奏**——热力图(周几 × 周次),对应格子亮起表示当天有复盘

无数据/数据不足时,分区仍保留结构,图表位置显示占位文案。

分区之间必须有明显视觉间距(不小于 1rem)。

## 4. "睡眠" tab

### 4.1 空状态(手环未接入,默认)

- 顶部一句话:"接入小米手环后,这里会显示你的睡眠分析"
- 底部灰化占位图(时长趋势 + 入睡时间分布),让用户预期看到什么
- 不显示"记录睡眠"按钮

### 4.2 有数据状态(接入手环后)

- 睡眠时长趋势(过去 30 天折线图,同"趋势"子 tab 中的睡眠图)
- 平均入睡/起床时间(两个数字 + 24 小时时间条可视化)
- 睡眠规律性(一个评分或标签:"规律 / 不太规律")
- **实践关联轻量观察**:如果最近某轮实践中睡眠时长明显下降(超过 X% 阈值),顶部出现一条 AI 观察卡(格式见第 7 节)

### 4.3 不做

- 手动录入表单
- 深睡/浅睡分期数据(等手环接入再看)
- 每天回顾 emoji 打卡类交互

## 5. "复盘" tab

沿用现有 `reflection-view.tsx` 的信息架构,但把"写今日复盘"从当前页展开改为**跳转独立详情页**。

### 5.1 主页面(paneReflection)

自上而下 3 段:

1. **月历卡** — r-card 风格月历:月切换按钮 + 月标题 + 7 列日期网格
   - 有复盘的日子在数字下加小圆点
   - 今天用描边圈起 + 高亮点(glow 色)
   - 点击某日跳转到该日复盘详情/阅读态

2. **写今日复盘 CTA** — 大按钮跳转到 `/records/reflection/today`(独立详情页,不在本页展开)

3. **历史复盘卡片列表** — 每张卡显示:
   - 日期 + 心情 emoji
   - "最满意/改进/明日/备注" 4 个字段行(非空的才显示)
   - 完全没有字段时显示"当天只记录了心情"

### 5.2 今日复盘详情页(records-reflection-detail.html 对应的独立路由)

- 顶部返回栏 + "自动保存"状态
- 大日期头(Noto Serif SC + kicker eyebrow)
- 心情卡:3 个大按钮(great→success 色、fair→glow 色、poor→danger 色)
- 4 个独立字段卡:最满意 / 改进 / 明日 / 备注
- 底部固定保存栏:显示"N 项已填 · 未保存" + 主按钮

### 5.3 不做

- 强制每天必填的表单流程
- 在主页面里直接展开编辑器(改为独立页)

**关于检查点自动汇总:** 检查点相关的自动汇总内容已移到"记录"tab 的实践详情页(时间线内),不再在复盘 tab 出现。

## 6. "回顾" tab

### 6.1 默认落地:最新周回顾

**顶部标题区:**
- 第 X 周回顾 + 日期范围(如"第 29 周回顾 · 7月13日–19日")
- 生成时间戳
- 若用户已确认过,显示"已确认";若还未看过,显示"未读"

**内容分区:**

1. **数据摘要**
   - 本周实践完成率
   - 本周复盘条数
   - 睡眠平均时长
   - 无叙述,只是数字

2. **AI 观察(1-3 条)**
   - 每条格式:观察句 + 依据 + 不确定性提示 + 三态按钮(见第 7 节)

3. **建议**
   - 是否需要新一轮实践 / 调整方法
   - 每条带三态按钮:采纳 / 稍后决定 / 忽略

### 6.2 历史周回顾

- 最新周回顾下方是历史列表,每条一行摘要(周次 + 观察数)
- 点击展开完整回顾
- 用户可以对历史回顾中的观察做二次修正(handoff 明确"用户主要查看、确认或修正")

## 7. AI 观察展示规范(贯穿全页)

不管在**睡眠、复盘、回顾**哪个 tab,AI 观察都遵循同一组件格式:

```
[观察] 最近 4 次"没开始",有 3 次提到工作疲惫。

[依据] 依据:第 2 轮实践 5 条记录 · 样本较小 · 只是相关,并非原因

[操作] [确认,值得看] [需要更多数据] [忽略]
```

关键约束:
- **观察句不含因果结论**——只描述相关性(如"最近 4 次…有 3 次…")
- **依据必须显示样本量**——"5 条记录"、"2 周数据"等
- **必须显示不确定性提示**——"样本较小"、"可能受季节影响"等
- **三态按钮不可省**——用户不确认时,AI 观察不进入个人结论/方法体系

组件在设计上应能复用(同一个 React 组件,不同 tab 引用)。

## 8. 数据流

### 8.1 数据来源(现有 schema,不改)

| 数据 | 来源 |
|------|------|
| 实践记录 | 每日轻量记录 + 详细记录(现有 habit_occurrence 表扩展,或新建 practice_log 表) |
| 睡眠 | 未来接入小米手环 API,当前无数据 |
| 复盘 | 每日一条 reflection 记录 |
| AI 观察 | 后端定时任务生成,存 ai_observation 表(带证据引用) |
| 周回顾 | 后端周日凌晨生成,存 weekly_review 表 |

### 8.2 用户交互对数据的影响

- 用户对 AI 观察点"确认" → observation 状态变为 confirmed,可被引用为方法证据
- 用户点"忽略" → observation 状态变为 dismissed,不再展示
- 用户点"需要更多数据" → observation 状态变为 pending_more,下一次生成时优先补充

## 9. 视觉与交互约束(继承 handoff 第 12 节)

- 沿用现有 rhythm 冷调 token(`--r-*`),暗色默认,4 皮肤可切
- 每个 tab 内部结构不同(时间线 / 图表 / 长文档 / 编辑器),避免"标题+卡片"同构
- 无全局悬浮加号
- 睡眠不做长表单(handoff 明确)
- AI 观察格式严格统一(见第 7 节)

## 10. 不在本次范围

- 阅读模块——独立顶级 tab,单独走 spec
- 情侣模块——已删除
- 运动 tab——移除
- 手环数据接入——工程实现在其他 spec
- 目标模块——handoff 说"默认从议题与实践中形成",归入计划页
- 移动端主题化 / 分享 / 导出——后续版本

## 12. 打开的问题(需在实现前决定)

1. 实践详情页是新路由还是模态展开?→ 推荐新路由 `/records/practice/[id]`,可分享/回退
2. 复盘的检查点汇总:后端预生成还是前端实时聚合?→ 推荐后端预生成,减少客户端计算
3. AI 观察数据模型:是否加"来源实践 id"字段?→ 是,用于跳转到证据源
4. 趋势图 90 天数据量:是否需要虚拟滚动/懒加载?→ 90 天级别不需,300 天以上再考虑
