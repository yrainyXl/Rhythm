# Rhythm V2 设计讨论记录 #5

**日期：** 2026-06-28
**讨论主题：** 入口层的核心定位 + 场景化交互设计

---

## 一、入口层的重新定位

### 问题
之前的入口层设计是「问问题 → 点按钮 → 跳转页面」，这变成了纯负担——凭空多了操作步骤。

### 新的定位
**入口层不是看门人，是快速的行动表面。** 用户打开 App，在入口层就能完成一件完整的事（5-15 秒），然后可以关掉走人，也可以自然下沉到骨架层。

### 入口层 vs 骨架层
- **入口层：** 「我今天快速记录一下就走」——轻、快、不需决策
- **骨架层：** 「我想翻一翻、看一看、规划一下」——深入浏览和规划

### 入口层能做的事
1. **完成最小行动** — 实践的最小行动直接在入口层完成，不跳转
2. **快速感受记录** — 三个表情或一个滑动，3 秒完成
3. **每日收尾** — 晚上打开，一个动作关闭今天
4. **引导对话创建方向/实践** — 整个流程在入口层完成，不下沉到表单页

## 二、场景化交互设计

### 场景 A：早上打开 + 有进行中实践

**画面：**
背景是清晨冷色调（淡蓝灰渐入暖白），整个画面非常空，只有中间几样东西。

**从上到下：**
- 顶部极小的文字，今天的日期和星期（近透明，不抢注意力）
- 中央略上方，一个呼吸中的圆形图标——代表今天的实践（比如早起）
- 圆形下方，一行大字：「第 4 天」
- 再下面，一行小字——今天的最小行动描述：「不看手机 3 分钟即可」

**交互：**
- 手指按下那个圆 → 圆开始缓慢填充，像水倒进杯子。3 秒填满。
- 填满瞬间，圆变成柔和的完成色（淡绿），一行字浮现：「做到了。」
- 整个过程没有按钮，没有确认框，没有跳转。
- 完成后 2 秒，画面可以：
  - 用户关掉 App（已经完成了今天的核心动作）
  - 用户向下滑动 → 平稳过渡到骨架层「今天」页面

**如果用户不想做：**
- 手指向左滑 → 圆轻轻飘走 → 一行小字：「没关系，今天还是你的。」
- 画面保持在这个状态，不再追问。用户可以向下滑进入骨架层。

**补充：辅助实践**
- 如果今天还有其他实践的待触发动作，主圆下方有一个很淡的小点，点它展开辅助实践的快速记录卡片。默认不展开，避免压迫感。

### 场景 B：晚上打开 + 有未记录实践

**画面：**
背景是暖色调（深蓝渐入暖橙），比早上更暗、更柔和。

**中央：**
- 一个半透明的圆，里面是今天实践的图标
- 下方一行字：「今天的那条实践，做了吗？」

**交互：**
- **右滑：** 「做到了」→ 圆亮起 → 短暂停留 → 显示「第 X 天完成」→ 消退
- **左滑：** 「今天没有」→ 圆变淡 → 出现一行小字：「没关系，没有对错。」
- **不上滑也不下滑：** 长按圆 → 弹出三个感受表情（😊 😐 😞）→ 选一个 → 结束
- **向下滑动：** 进入骨架层，不做入口层操作

**完成后的自然衔接：**
如果是周期最后一天（第 7 天），完成记录后圆不消退，而是变成一段更深的引导：
> 「这一轮结束了。想花一分钟想想结论吗？」
- 「现在想」→ 进入入口层内的轻复盘对话（见场景 E）
- 「晚点再说」→ 关闭，第二天会再次提醒

### 场景 C：早上/中午打开 + 没有进行中实践

**画面：**
更空。没有圆，只有中央一行字。

**轮换文案（每次打开略有不同）：**
- 「今天感觉很安静。」
- 「没有正在进行的事情。也挺好的。」
- 「想到什么想试的吗？」

**交互：**
- **点那行字** → 出现三个轻选项，不是按钮，是三行淡色的字：
  - 发起一个想试的方向
  - 去花园看看
  - 就这样
- **「就这样」** → 3 秒后画面自动消退，进入骨架层的「今天」页
- 其他两个选项 → 分别进入对应的引导对话（见场景 E）

### 场景 D：刚完成一个实践周期 + 等待复盘

**画面：**
背景略有变化——有一点点仪式感但不隆重。色调从日常色变成一个过渡色。

**中央文案：**
- 「早起不看手机」
- 「7 天结束了。」
- 下方一个微微发光的圆点

**交互：**
- 点那个发光点 → 进入轻复盘对话
- 右滑 → 「稍后再说」
- 如果连续跳过复盘超过 3 天 → 圆的发光变亮，但不追问

**轻复盘对话（场景 E）：**
不是表单。是三次轻问答。每次屏幕上只出现一个问题，用户回应后滑入下一个。
1. 这个方法对你有用吗？—— 三个选项横排：「有用」「部分有用」「没感觉」
2. 什么时候你更容易做到？—— 一个可以自由写字的小区域，可以不填
3. 下一步你想怎么做？—— 「一直保持」「再试一轮调整」「先放一放」

三问结束后，一行确认文字：「记住了。这条实践已经加入你的方法地图。」画面自然沉降到「内心的地图」页面，新碎片已经在对应区域亮着。

### 场景 F：发起方向（深度引导对话）

**触发方式：**
- 从场景 C 的「发起一个想试的方向」进入
- 或者从「内心的地图」页面的「新的方向」入口进入

**画面：**
背景完全沉浸——不是表单弹出，是整个画面变成对话空间。

**对话流程（5-6 步，每步一屏）：**

**第 1 屏：** 空屏。停顿 2 秒。
> 「你现在最想改变的是什么？」

下方出现一个干净的输入区域——没有边框、没有标签、没有字符计数。用户自己决定写多长。写完停顿 3 秒，文字缓缓上移，进入下一屏。

**第 2 屏：** 
> 「为什么是现在？」

不是追问，是引导用户把这个方向和「此刻的我」建立连接。同样一个安静输入区。可以不填。

**第 3 屏：**
> 「如果这个方向实现了，你的生活会有什么不一样？」

这次回答不管长短，结束后，系统把前三屏的关键词聚合成一个方向摘要（用本地规则提取，或者就是用户原样的表述）。

**第 4 屏：**
> 「好，我记住了。这个方向会从这里开始。」

> [用户的方向名称]

> 「接下来，你可以随时在这个方向下发起点小实践。」

确认完成，方向已创建。画面缓慢沉降到「内心的地图」——新方向已经在星图/地图正中微微发光。

整个流程没有「提交」按钮，没有表单验证，没有「请填写必填字段」。用户可以在任何一步停下来关掉 App，方向仍然保存。

## 三、所有场景总结

| 场景 | 触发 | 核心交互 | 完成时间 | 可下沉到 |
|------|------|---------|---------|---------|
| A: 有实践+早上 | 时间+实践状态 | 长按圆完成最小行动 | 3-5秒 | 今天 |
| B: 有实践+晚上 | 时间+实践状态 | 左右滑完成记录 | 2-3秒 | 今天 |
| C: 无实践 | 无进行中实践 | 点文案选方向 | 5-10秒 | 今天/花园 |
| D: 等待复盘 | 实践刚结束 | 点发光点进入轻复盘 | 10-15秒 | 内心的地图 |
| E: 轻复盘 | D触发 | 三次轻问答 | 15-20秒 | 内心的地图 |
| F: 深度引导 | C触发/地图入口 | 5-6屏对话创建方向 | 30-60秒 | 内心的地图 |
